# 角色形象重生成建议

## 结论

当前公开信息不足以证明你提到的那张“终末地管理员企鹅”参考图，究竟是由哪个精确模型、哪一个工作流生成的。能查到的是社区传播里至少存在两类路线：

- 有创作者直接把类似风格内容标注为“即梦AI”生成。
- 也能看到围绕《终末地》审美的社区 LoRA / 风格化模型页，说明 `SDXL / Pony / LoRA` 也是常见路线。

这只能说明“可能的生成路线”，不能证明某一张图的唯一来源。

## 为什么不建议直接复刻现成图

- 你要装到 Windows 桌面长期使用，最好避免直接复刻来源不明的 fanart。
- 《终末地》本身是明确 IP，直接照抄脸和服饰会带来版权和风格混淆问题。
- 更稳的做法是做一个“终末地工业控制感 + 管理员企鹅职责感”的原创桌宠。

## 推荐路线

### 路线 A：优先落地和 API 接入

用 OpenAI 的 `gpt-image-1` 生成原创形象草图，再手工挑一版进入后续切图 / Live2D / Spine。

适合原因：

- 后续能直接接 API，方便把“形象生成”“表情重绘”“宣传图生成”统一进一个后端。
- 风格控制虽然不如专门训练 LoRA 精细，但第一版原型最快。

### 路线 B：优先贴近社区二创审美

用 `SDXL / Pony` 底模叠加“终末地工业科幻 / 管理员制服 / 小动物拟人”方向的 LoRA 做多轮出图。

适合原因：

- 更容易打磨出“白灰蓝 + 冷色工业感 + 终末地系 UI 语言”的细节。
- 更容易做多张表情、动作分镜和立绘变体。

代价：

- 管理 LoRA、底模、采样参数、版权边界更麻烦。
- 如果没有稳定工作流，复现性会比 API 方案差。

## 建议的人设约束

不要直接要求“复制终末地管理员企鹅”。改成下面这类约束更安全：

- 小体型南极企鹅 / 帝企鹅幼体比例
- 工业管理站值守员
- 白灰主色，辅色为冰蓝、浅青、少量警示橙
- 胸前有访客证 / 管理员胸牌 / 小型终端
- 帽子或耳机带有控制中心标识
- 表情偏冷静、可靠、会提醒风险
- 造型简洁，适合后续拆成待机 / 聆听 / 思考 / 回复 / 警戒 五种状态

## 动作拆分建议

第一批只做 5 个状态，足够桌宠 UI 使用：

1. `idle`：轻微左右晃动，呼吸感
2. `listening`：头部前倾，耳机或终端亮起
3. `thinking`：低头看终端，光点流动
4. `speaking`：抬头张口，小幅手势
5. `guarded`：站直、胸牌高亮、警示橙点闪烁

## 与产品功能的绑定建议

- 形象只负责“陪伴 + 状态表达”，不要承载所有设置 UI。
- 聊天、动作、设置都做成可展开的小浮层，默认只显示宠物本体。
- 高风险桌面动作必须显式二次确认，不要让角色表情“误导为已执行成功”。

## OAuth / 权限设计建议

- AI 登录优先走标准 OAuth / OIDC 回跳，而不是在前端长期保存 Access Token。
- Windows 侧只暴露白名单动作，不开放自由命令执行。
- 敏感目录、浏览器数据、剪贴板、文件上传、账号信息读取，都必须单独确认。
- 所有桌面动作都要写入审计日志，默认最小权限。
