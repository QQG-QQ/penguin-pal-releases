# 明日继续任务（桌宠改造）

日期：2026-03-10

## 用户确认的目标

1. 支持明显的关闭方式
- 在桌宠窗口内提供可见的关闭按钮（不仅依赖托盘）。
- 关闭行为优先为隐藏到托盘；若用户选择退出再彻底退出进程。

2. 支持自然拖动
- 允许在桌宠主体区域直接拖动窗口。
- 减少“必须点小拖拽条”这类高门槛交互。

3. 缩小窗口尺寸
- 当前窗口过大。
- 调整为接近 QQ 宠物风格的小型悬浮尺寸与布局。

4. 默认极简模式
- 启动时仅显示宠物本体。
- 聊天/设置等大面板改为按需展开，不默认占屏。

5. 角色形象替换
- 目标形象：终末地女管理员企鹅。
- 处理当前“渲染不完整”的问题：
  - 资源加载失败回退策略
  - 图片/动画容器尺寸与裁切策略
  - 高 DPI 与透明背景兼容

## 实施顺序（建议）

1. 先修交互（关闭、拖动、默认极简、小窗尺寸）
2. 再接入角色资源并完善渲染回退
3. 最后统一视觉样式与状态动画

- `npx.cmd tauri dev`

- 默认只显示小桌宠

- 角色资源加载成功，失败时有降级显示
